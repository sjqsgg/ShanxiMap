# 山西古建筑访古地图 — 设计文档

日期：2026-07-12
状态：已经用户确认

## 一、产品定位

山西省古建筑访古地图网页应用，帮助访古旅行者发现不被大众熟知的山西古建筑。
纯粹古建筑热爱，不蹭游戏/影视IP。视觉概念：**国保档案馆**——每处国保天然带有
编号、批次、年代、公布年份，用「档案」作为贯穿全站的视觉与叙事语言。

参考视觉来源（用户提供）：
- virgilabloh.com：技术档案文件语言（等宽字体元数据表头、点状分隔线、巨大标题、
  滚动时档案文件依次滑出）→ 换成米色宣纸色调
- dreamrecorder.ai：磨砂玻璃/柔焦质感 → 用于地图页浮层

## 二、整体结构

两个页面，一条滚动叙事 + 一张全屏地图：

### 首页 `/` = 档案馆（滚动叙事）
1. 档案封面：等宽字档案表头（SHANXI_ARCHIVE / 日期 / NO.532）、点状分隔线、
   巨大衬线标题「山西访古档案」、统计数字（唐构15 · 宋金117 · 元126 · 明清147）
2. 向下滚动：21张「必去」档案卡依次滑出，带轻微随机旋转角度（散落卷宗感），
   每张卡含：档案编号、批次+年份、建筑名、访古等级章、朝代、区县、一句话看点、
   营造学社标注（如有）。点击卡片 → 跳转 `/map?id=xxx` 定位到该建筑
3. 滚动到底：磨砂玻璃过渡层，「进入完整档案」→ `/map`
4. 右上角固定「直接查阅地图 →」按钮，回访用户可跳过叙事

### 地图页 `/map` = 工作台（全屏地图）
- 全屏高德地图，自定义米色纸面样式底图
- 所有 UI 浮层用 backdrop-filter blur 毛玻璃 + 半透明纸色
- URL 状态可分享：`/map?id=87&dynasty=唐&city=忻州市...`

## 三、地图页 UI 细节

### 点位
- 必去：朱砂大点（#b03a2e），始终显示名称标签
- 推荐：赭石中点（#c87f41）
- 小众：墨灰小点（#a89a7e），放大后显示名称
- 可选（遗址/墓葬/革命史迹）：默认隐藏，筛选器可开启
- 营造学社考察过的建筑：点位加「测」字小印章角标

### 档案卡弹窗（点击点位）
- 等宽字编号表头（档案NNN · 第N批 · 年份）
- 衬线建筑名 + 访古等级章
- 朝代 / 区县 / 类型
- 简介（50-150字）
- 营造学社来源标注（如「营造学社详测 ·《记五台山佛光寺的建筑》1944」）
- 操作：复制坐标、跳转高德App导航

### 筛选（毛玻璃 chip，多选叠加）
- 朝代：唐 / 五代 / 辽 / 宋 / 金 / 元 / 明 / 清（含北宋归宋）
- 地市：11个
- 访古等级：必去 / 推荐 / 小众 / 可选
- 类型：古建筑 / 石窟寺及石刻 / 古遗址 / 古墓葬 / 近现代
- 营造学社足迹：开关
- 搜索框：按名称模糊搜索

### 列表侧栏
- 按档案编号排列，可搜索
- hover 列表项 ↔ 地图点位高亮联动
- 移动端收起为底部抽屉

## 四、数据管线

数据源：`shanxi_guobao_enhanced.json`（532条）→ 产出 `shanxi_guobao_final.json`
（前端唯一数据源）。步骤：

1. **营造学社标注**（按 FABLE_TASK_yingzao.md）：新增 `yingzao`（详测/到访/空）
   和 `yingzao_source` 两字段，仅标注文献确认的约18处；同时输出
   `shanxi_guobao_final.csv` 和 `yingzao_list.csv`
2. **已知坐标**：28个 WGS-84 坐标转 GCJ-02 后填入
3. **批量地理编码**：高德 Web服务 key，「名称+区县」查询；返回 adcode 与数据
   区县比对消歧；置信度分级：
   - HIGH：兴趣点级匹配且区县一致 → 直接入库
   - MEDIUM：道路/村镇级匹配且区县一致 → 入库并标记 `geo_precision: "approx"`
   - LOW：仅匹配到市县 → 用区县中心近似，标记 `geo_precision: "county"`，
     输出人工核查清单
4. **简介**：必去21 + 推荐49 逐条手写（位置+始建年代+核心看点，50-150字）；
   其余按「位置+朝代+类型+批次」模板生成基础简介，标记 `desc_source: "template"`
   便于后续增量补写
5. 坐标系统一 GCJ-02（与高德底图一致）

## 五、技术架构

- Next.js 15 (App Router) + TypeScript + Tailwind CSS
- 地图：高德 JS API 2.0（前端 key + 安全密钥，jsapi loader）
- 数据：final JSON 打包进构建产物，无后端
- 部署：Vercel
- 扩展空间：社区/账号/数据库后续以 API Routes + Postgres/Supabase + NextAuth
  加入，不改变现有结构

### 安全注意
- 前端 key + 安全密钥必然暴露在源码中（高德机制如此）：上线后需在高德控制台
  为前端 key 配置域名白名单
- Web服务 key 仅用于本地编码脚本，不进前端代码、不提交到仓库（.env 忽略）
