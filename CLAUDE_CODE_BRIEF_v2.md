# 山西古建筑地图 — Claude Code 工作说明 v2

## 项目目标
做一个山西省古建筑访古地图网页应用，帮助用户发现不被大众熟知的山西古建筑，方便实地访问。

## 产品调性
- 纯粹的古建筑热爱，不蹭任何游戏/影视IP
- 面向想深入山西的访古旅行者

**推荐方案：高德地图地理编码API**
- 申请：https://lbs.amap.com （免费额度5000次/天）
- 接口：`https://restapi.amap.com/v3/geocode/geo?address=NAME+ADDRESS&city=shanxi&key=YOUR_KEY`

**坐标获取策略**：
1. 先用上面28个已知坐标直接填入
2. 高德API用"名称+地址"批量查询剩余504个
3. 按置信度分三档：HIGH（景区POI匹配）/ MEDIUM（道路级匹配）/ LOW（仅匹配到市县）
4. HIGH直接入库，MEDIUM+LOW单独输出人工核查
5. 坐标系统一存GCJ-02（高德原生），前端用高德JS SDK

**已知风险**：
- 约20-30条地址只到市级（如"山西省运城市"），需补全到村镇
- 同名建筑混淆（多个"玉皇庙""关帝庙"）
- GCJ-02和WGS-84偏差约500m，已有的28个坐标是WGS-84，需转换

### Task 2：批量获取建筑简介

JSON中description字段为空，需要补充。

**来源优先级**：
1. 维基百科（有结构化摘要，覆盖约100-150个知名建筑）
2. 百度百科（覆盖最广，基本每个国保都有词条）
3. 山西省文旅厅 wlt.shanxi.gov.cn（官方简介）

**简介规范**：
- 长度：50-150字，适合地图弹窗显示
- 内容：位置 + 始建年代 + 核心看点（1-2个）
- 示例："位于五台县豆村镇佛光村。唐大中十一年（857年）重建，现存最大唐代木构建筑。集唐代建筑、塑像、壁画、墨书题记四绝于一身，被梁思成称为'中国第一国宝'。"

### Task 3：前端地图应用

**技术栈**：
- 框架：Next.js 14（App Router）
- 地图：高德地图 JS API 2.0（中国坐标系最准）
- 数据：直接引用 shanxi_guobao_enhanced.json
- 部署：Vercel

**核心功能（MVP）**：
1. 地图上显示所有古建筑点位，按访古等级用不同颜色/大小区分
2. 点击标记弹出卡片：名称、朝代、所在地、国保批次、简介、访古等级
3. 侧边栏列表，支持搜索
4. 筛选器：
   - 朝代（唐/宋/金/元/明/清）
   - 地市（11个）
   - 访古等级（必去/推荐/小众）
   - 类型（古建筑/石窟/全部）

**地图标记样式建议**：
- 必去：大红点，始终显示名称
- 推荐：中橙点
- 小众：小灰点，缩放后显示

**后续功能**：
- 路线规划（同一地市的古建筑串联成访古线路）
- 朝代时间轴（滑动时间轴，地图上只显示对应朝代的建筑）
- 用户贡献（补充照片、开放时间、交通方式）
- 小红书分享卡片生成
