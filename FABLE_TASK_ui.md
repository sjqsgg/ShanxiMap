# 山西访古档案 — UI/UX 改造指令

## 项目现状
Next.js应用，localhost:3000，高德地图JS API，已有基础地图和档案卡片样式。
视觉调性已定：米色底、档案风格、等宽字体细节、"必去"印章、营造学社标签。

---

## 总体结构

```
首页 (/)
├── Hero区：大标题 + 统计数字
├── 档案袋动画（scroll触发展开）
└── 展开后：地图 + 索引列表

地图交互：
├── 点击标记点 → 右侧预览抽屉滑入
└── 抽屉顶部"查看完整档案" → 跳转详情页

详情页 (/site/[id])
└── 独立页面，完整信息
```

## 改造任务一：地图修复

### 1.1 锁定山西省范围
```javascript
// 高德地图限制边界
map.setLimitBounds(new AMap.Bounds(
  [110.23, 34.58],  // 西南角
  [114.56, 40.74]   // 东北角
))
map.setZooms([7, 14])  // 最小7（看全省），最大14（县城级别）
```

### 1.2 加载山西行政区划边界
- 用高德DistrictSearch加载山西省+11个地市的行政边界
- 边界线样式：`strokeColor: '#8B7355'`，`strokeWeight: 1`，`strokeOpacity: 0.4`，`fillOpacity: 0`
- 省界线稍粗：`strokeWeight: 2`，`strokeOpacity: 0.6`

```javascript
const districtSearch = new AMap.DistrictSearch({
  subdistrict: 1,
  extensions: 'all',
  level: 'city'
})
districtSearch.search('山西省', (status, result) => {
  // 绘制边界polygon
})
```

### 1.3 标记点样式
按访古等级区分：
- 必去：直径16px，`#8B1A1A`深红色，始终显示建筑名label
- 推荐：直径10px，`#8B5E3C`棕橙色，hover显示名称
- 小众：直径6px，`#999990`灰色，zoom≥11时显示名称
- 石窟寺及石刻：在对应等级颜色基础上加菱形而非圆形

## 改造任务二：首页结构重建

### 2.1 Hero区（保留现有内容，加入动画）

用React Bits的`BlurText`组件：
```bash
npx jsrepo add https://reactbits.dev/default/TextAnimations/BlurText
```
- "山西访古档案"大标题用BlurText，文字从模糊渐清入场
- 统计数字（532/13/132/126/16）用CountUp动画，进入视口时触发

### 2.2 档案袋动画（scroll-driven，Framer Motion）

Hero区底部放一个档案袋图形，用CSS绘制：
- 档案袋主体：米黄色矩形，圆角，顶部有封口盖
- 封口盖：独立元素，有绳扣细节

scroll触发逻辑（用Framer Motion的`useScroll` + `useTransform`）：
```javascript
const { scrollYProgress } = useScroll()

// 封口盖旋转打开
const lidRotate = useTransform(scrollYProgress, [0.1, 0.3], [0, -110])

// 内容从档案袋里展开
const contentScale = useTransform(scrollYProgress, [0.25, 0.5], [0, 1])
const contentY = useTransform(scrollYProgress, [0.25, 0.5], [100, 0])
```

档案袋展开后显示内容区（地图+索引列表），用`AnimatePresence`控制。

### 2.3 地图区

档案袋展开后的第一屏：
- 占满视口高度（100vh）
- 顶部保留筛选栏：朝代筛选（全部/唐/宋/金/元/明/清）+ 地市筛选（全部/11个地市）+ 类型筛选（全部/古建筑/石窟）+ 访古等级（全部/必去/推荐/小众）
- 筛选栏样式：等宽字体，细线分隔，与整体调性一致

### 2.4 索引列表区

地图正下方，按**地市**分组：

```
运城市 · 102处                          [展开/收起]
─────────────────────────────────────
永乐宫  解州关帝庙  广仁王庙  万荣东岳庙  芮城城隍庙
后稷庙  泛舟禅师塔  清凉寺  ···

```

样式参考你截图里"中国古代才女"的布局：
- 左侧地市名+数量，右侧是建筑名平铺
- 建筑名之间用空格分隔，不用bullet
- 建筑名可点击，点击后地图飞到该点并打开预览抽屉
- 必去建筑名加红色下划线区分

地市顺序按国保数量降序：运城→长治→晋城→晋中→临汾→吕梁→忻州→太原→大同→阳泉→朔州

用React Bits的`FadeContent`，每个地市组滚动进入视口时淡入：
```bash
npx jsrepo add https://reactbits.dev/default/Animations/FadeContent
```

## 改造任务三：预览抽屉

点击地图标记点或索引列表里的建筑名触发。

### 3.1 动画
用Framer Motion `AnimatePresence`从右侧滑入：
```javascript
<motion.div
  initial={{ x: '100%' }}
  animate={{ x: 0 }}
  exit={{ x: '100%' }}
  transition={{ type: 'spring', damping: 30, stiffness: 300 }}
  className="preview-drawer"
/>
```
地图同时向左偏移抽屉宽度（约30vw）。

## 动效总结（给Fable安装的依赖）

```bash
# Framer Motion（可能已装）
npm install framer-motion

# React Bits组件（按需安装）
npx jsrepo add https://reactbits.dev/default/TextAnimations/BlurText
npx jsrepo add https://reactbits.dev/default/Animations/FadeContent
```

所有动画原则：
- 时长控制在300-500ms，不要太慢
- 用spring而非linear，感觉更自然
- 移动端降低动画复杂度
