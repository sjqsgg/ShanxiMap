# 03 — 完成严格的地点档案运行时校验

**要实现的内容：** 数据维护者可以把未知输入交给唯一的纯校验边界，并一次获得稳定、可定位的全部问题；合法数据被转换为可信地点档案，非法字段和值会在命令行验收中被拒绝。

**前置任务：** 02 — 从运行时地点档案中删除访古等级

**状态：** ready-for-agent

- [ ] 公共 validator 接收 `unknown`，成功时返回完整可信的地点档案集合，失败时返回按确定规则排列的问题列表。
- [ ] 正式运行时记录及图片对象拒绝未知字段，因此重新出现的 `tier` 和拼错的字段均会失败；source 与 intermediate 数据不受此规则限制。
- [ ] 集合形态、非空集合、唯一正整数 ID、必填与可空字段、字符串与整数约束均符合已批准 spec，且不固定 532 条或要求 ID 连续。
- [ ] 城市、遗产类型、简介来源和坐标精度使用严格枚举；朝代、地址、县区、批次编号和坐标来源等展示或来源文字保持开放。
- [ ] 坐标必须是有限数字并落在记录于事实文档的宽松山西范围内；错误信息明确说明范围通过不代表坐标准确。
- [ ] 营造学社描述与来源必须成对出现；Wikipedia 和图片 URL、图片许可与署名满足已批准的跨字段约束。
- [ ] `rating`、`tel`、`is_open` 仅兼容当前已批准形态，包含对字符串评分和空数组评分的显式过渡测试，不借此设计最终实用信息 schema。
- [ ] 测试覆盖有效完整记录、兼容空值、错误 primitive、未知字段、重复或非法 ID、越界与写反坐标、非法枚举、URL、图片和营造学社配对。
- [ ] 命令行继续调用同一个 validator；真实 artifact 通过，人工非法输入产生可定位输出和非零退出码。
- [ ] 本地聚合检查与生产构建通过。
