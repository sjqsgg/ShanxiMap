# 01 — 移除访古等级对浏览体验的影响

**要实现的内容：** 访客不再通过“必去、推荐、小众、可选”等价值等级发现或判断地点档案；地图、筛选、详情和建筑总索引改用中性的浏览规则，同时保持现有页面入口和地图交互可用。

**前置任务：** 无——可以立即开始

**状态：** ready-for-agent

- [ ] 首页地图与地图直达模式不再提供访古等级筛选，旧的等级 URL 参数不再影响筛选结果或成为可分享状态的一部分。
- [ ] 地图图例、预览和详情中不再出现访古等级文字或徽章。
- [ ] 所有地点档案的地图点具有相同的基础视觉重量；选中、悬停、缩放和碰撞处理仍可提供必要的交互反馈。
- [ ] 不新增 `famous`、`featured`、推荐分数或其他替代等级；省级全览常显名称明确留给后续响应式地图配置。
- [ ] 建筑总索引继续按城市分组，并在每个城市内按年代从早到晚排列，再以名称和稳定 ID 确定同年代顺序。
- [ ] 公开筛选、URL 和索引排序行为有聚焦测试，且本地聚合检查与生产构建通过。
- [ ] 本阶段暂时保留运行时数据和领域类型中的旧 `tier` 字段，但产品代码不再读取或展示它，为下一阶段安全删除创造条件。
