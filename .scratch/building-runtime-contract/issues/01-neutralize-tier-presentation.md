# 01 — 移除访古等级对浏览体验的影响

**要实现的内容：** 访客不再通过“必去、推荐、小众、可选”等价值等级发现或判断地点档案；地图、筛选、详情和建筑总索引改用中性的浏览规则，同时保持现有页面入口和地图交互可用。

**前置任务：** 无——可以立即开始

**状态：** complete

- [x] 首页地图与地图直达模式不再提供访古等级筛选，旧的等级 URL 参数不再影响筛选结果或成为可分享状态的一部分。
- [x] 地图图例、预览和详情中不再出现访古等级文字或徽章。
- [x] 所有地点档案的地图点具有相同的基础视觉重量；选中、悬停、缩放和碰撞处理仍可提供必要的交互反馈。
- [x] 不新增 `famous`、`featured`、推荐分数或其他替代等级；省级全览常显名称明确留给后续响应式地图配置。
- [x] 建筑总索引继续按城市分组，并在每个城市内按年代从早到晚排列，再以名称和稳定 ID 确定同年代顺序。
- [x] 公开筛选、URL 和索引排序行为有聚焦测试，且本地聚合检查与生产构建通过。
- [x] 本阶段暂时保留运行时数据和领域类型中的旧 `tier` 字段，但产品代码不再读取或展示它，为下一阶段安全删除创造条件。

- 聚焦测试先暴露旧的等级优先排序和缺失的纯 URL seam；实现后，旧 `tier` 查询被忽略且不再序列化、不同旧等级的匹配档案同时保留、城市内按年代/名称/ID 排序。
- 浏览器检查覆盖带旧等级参数的 `/map`、筛选面板、地图预览和档案详情；等级参数从 URL 消失，等级控件、徽章与印章均不存在，地图使用中性基础点位，浏览器控制台为 0 errors。
- `npm run check`：通过；ESLint 为 0 errors、6 个既有 warnings，TypeScript 通过，11 项测试通过，532 条运行时记录通过校验。
- `npm run build`：通过；成功生成 537 个静态页面。
- 运行时 JSON、`Building.tier`、基础 validator 与数据管线脚本未修改，留待工单 02 收缩契约。
